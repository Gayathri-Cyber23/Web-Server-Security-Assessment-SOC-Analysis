# Phase 3: SOC-Style Analysis Summary

## Purpose

Phase 3 focused on understanding how security events are monitored and analyzed.

The goal was to review system activity, identify unusual behavior, and gain basic exposure to how Security Operations Center (SOC) teams investigate potential threats.

## Work Completed

- Performed basic vulnerability scanning and service enumeration  
- Reviewed open ports and running services  
- Identified outdated software and weak configurations  
- Observed web server behavior during scanning activity  
- Reviewed basic logs to understand system activity  

## Security Concepts Applied

- Log analysis basics  
- Event monitoring  
- Vulnerability identification  
- Threat awareness  
- System activity tracking  

## Tools and Methods

- Nmap (basic scanning)  
- Nikto (web server checks)  
- Linux command line  
- Basic log review  

## What I Learned

- How security tools generate activity that can be observed in logs  
- How system behavior changes during scanning or probing  
- Why monitoring is important for detecting suspicious activity  
- How SOC teams use logs and alerts to understand potential threats  

## Explanation

In Phase 3, I focused on understanding how system activity can be monitored and analyzed. I reviewed how basic scanning tools interact with a system and how those actions can be observed through logs. This gave me exposure to how SOC teams detect and investigate potential security events.
